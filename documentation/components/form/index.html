Form Component Documentation

This page provides an overview of the Form component in the KBrandt Design System. Forms are essential for collecting user input and are designed to be accessible, customizable, and easy to use.

  • Purpose: Collect user input in a structured and accessible way.
  • Features: Includes text inputs, text areas, dropdowns, checkboxes, and radio buttons.
  • Accessibility: Built with semantic HTML and ARIA attributes for better usability.
