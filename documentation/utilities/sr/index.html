Screen Reader Only

The screen reader only (sr-only) class is used to hide elements visually while keeping them accessible to screen readers, enhancing accessibility for users with visual impairments.

Table of Contents

  1. Usage
  2. Styling
  3. Example

Usage

The `.sr-only` class is applied to elements that should be hidden from visual presentation but remain accessible to screen readers. This is useful for providing additional context or labels that are not necessary for sighted users.

Example

This text is hidden but accessible to screen readers.

Explanation

  • clip: Uses the `rect(0 0 0 0)` value to hide the element visually.
  • clip-path: Uses `inset(50%)` to ensure the element is not visible.
  • height & width: Set to 1px to maintain layout without showing the element.
  • overflow: Set to hidden to prevent content from being displayed.
  • position: Set to absolute to remove the element from the document flow.
  • white-space: Set to nowrap to prevent text from wrapping.
  • :not(:focus):not(:active): Ensures the element becomes visible when focused or active, allowing screen reader interaction.

Styling

The styling for `.sr-only` ensures that the element is hidden visually but can still be accessed and interacted with by screen readers.

CSS Properties

  • clip: Hides the element visually.
  • clip-path: Provides an additional layer of visual hiding.
  • height & width: Minimizes the element's dimensions.
  • overflow: Prevents any content from overflowing the hidden area.
  • position: Removes the element from the normal document flow.
  • white-space: Prevents text from wrapping, ensuring it stays within the hidden area.

Example

HTML

<div class="sr-only" tabindex="0">
    This text is hidden but accessible to screen readers.
</div>

CSS

.sr-only:not(:focus):not(:active) {
    clip: rect(0 0 0 0);
    clip-path: inset(50%);
    height: 1px;
    overflow: hidden;
    position: absolute;
    white-space: nowrap;
    width: 1px;
}
